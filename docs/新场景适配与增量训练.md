### 目标
当更换到“种蛋更大/颜色不同/光照不同”的新场景后，原模型可能出现漏检，导致跟踪 ID 频繁丢失，进而蛋-笼位匹配不稳。

本仓库已基于 **YOLOv8 + OC-SORT**，推荐优先按“调参 → 轻量预处理 → 增量微调”的顺序落地，成本最低、收益最大。

---

### 一、先做无需重训的稳健化（建议先跑一轮对比）

#### 1) 启用 BYTE 的二次关联（补跟踪）
你需要同时满足两点：
- `configs/config.yaml`：`conf` 设为 **0.1~0.2**（放行低分检测）
- `configs/ocsort.yaml`：`det_thresh` 设为 **0.3**（主匹配与新建轨迹使用 >0.3；0.1~0.3 仅用于补跟踪，不新建）

这样即便新场景蛋的置信度下降，OC-SORT 也可以用低分框把轨迹“续上”，整体更稳。

#### 2) 可选：开启轻量预处理（对光照/颜色偏移更稳）
`configs/config.yaml` 增加了：
- `preprocess.enabled`
- `preprocess.clahe.enabled`
- `preprocess.gamma`

建议顺序：
- 先开 `clahe`（对暗/反光更稳）
- 再微调 `gamma`（1.2~1.8 逐步尝试）

---

### 二、增量标注与微调训练（推荐方案：不是从零重训）

#### 1) 抽帧构建数据集骨架
示例（从新场景视频抽帧到 `data/dataset_eggs`）：

```bash
python tools/prepare_dataset.py --input-dir data/capture/camera_0 --videos 20251015_083940.mp4 --output-dir data/dataset_eggs --fps 3 --split
```

#### 2) 伪标签（自动标注）加速人工校正
用现有权重先生成伪标签（再去 CVAT/LabelImg 等工具里“校正”）：

```bash
python tools/auto_label_yolov8.py --model runs/detect/eggs_train2/weights/best.pt --source data/dataset_eggs/images/train --out-labels data/dataset_eggs/labels/train --imgsz 896 --device 0 --conf-egg 0.15 --conf-qr 0.25
```

说明：
- 该脚本输出的是 **YOLO 检测格式**（`cls xc yc w h`，归一化到 0~1）
- 你可以把生成的标签导入标注工具做“纠错/补标”，效率比纯手工高很多

#### 3) 若你的标签来自 CVAT Track 导出（6列），先修复成 YOLO 5列

```bash
python tools/fix_yolo_labels.py --root "data/dataset_eggs"
```

#### 4) 训练（微调）
建议直接在现有最优权重上微调，而不是从 `yolov8s.pt` 重新开始：

```bash
python tools/train_yolov8.py --data data/dataset_eggs/data.yaml --imgsz 896 --epochs 80 --batch 16 --device 0 --model runs/detect/eggs_train2/weights/best.pt --name eggs_finetune_new_scene
```

产物位于：
- `runs/detect/eggs_finetune_new_scene/weights/best.pt`

#### 5) 上线使用
修改 `configs/config.yaml`：
- `modelPath` 指向新生成的 `best.pt`
- 保留或按需调整 `conf/imgsz` 与 `preprocess`

---

### 三、经验值（给你一个“默认可用”的起点）
- **conf**：0.1（先确保召回；后续用匹配稳定帧去抖）
- **imgsz**：896 或 960（算力允许可适当上调）
- **det_thresh(ocsort)**：0.3（启用补跟踪）
- **preprocess**：先开 CLAHE，再微调 gamma
